## 1. 서비스 개요

| 항목 | 내용 |
|------|------|
| **서비스명** | LearnAI |
| **서비스 유형** | AI 기반 학습 관리 시스템 (LMS, Learning Management System) |
| **배포 URL** | https://learnai-woad.vercel.app |
| **핵심 가치** | "AI가 교육의 모든 과정을 스마트하게 지원한다" |
| **대상 사용자** | 온라인 강의를 운영·수강하는 교사 및 학생, 교육 운영자 |

LearnAI는 **Anthropic Claude AI**를 교육 현장에 깊이 통합한 한국어 웹 기반 학습 관리 플랫폼입니다. 교사는 강의 자료를 업로드하면 AI가 자동으로 퀴즈를 생성해주고, 학생은 AI 튜터와 실시간 대화하며 학습하고, 운영자는 AI 분석 리포트로 이탈 위험 학생을 선제적으로 파악할 수 있습니다.

## 2. 기획 배경 및 문제 정의

### 2-1. 현재 교육 현장의 문제점

#### 교사 측면
- 강의 자료를 바탕으로 퀴즈를 직접 출제하는 데 **평균 2~3시간** 소요
- 수강생 진도와 이해도를 파악하기 위한 **별도 분석 도구 부재**
- 수십 명의 코딩 과제를 개별 채점하고 피드백을 주는 데 **과도한 시간** 필요

#### 학생 측면
- 강의 시간 외 궁금증 해결을 위한 **즉각적 답변 채널 부재**
- 자신의 학습 수준에 맞는 **개인화 학습 경로 부재**
- 코딩 과제 제출 후 구체적인 **개선 피드백 부재**

#### 운영자 측면
- 수강생 학습 현황을 직관적으로 파악할 수 있는 **통합 대시보드 부재**
- 강의 이탈 위험 학생을 **사후에야** 파악하는 문제

### 2-2. 시장 기회

- 국내 이러닝 시장 규모: 연간 5조원 이상 (2024년 기준)
- 생성형 AI의 교육 분야 적용 급증 — EdTech + AI의 결합 수요 폭발적 증가
- 기존 LMS(Moodle, Canvas 등)는 AI 기능이 매우 제한적이거나 별도 비용 발생

## 4. 사용자 분석 및 페르소나

### 역할별 사용자 구성

```
┌─────────────────────────────────────────────────┐
│                   LearnAI 사용자                  │
├───────────────┬────────────────┬─────────────────┤
│    학생(Student)│  교사(Teacher) │  운영자(Admin)   │
│               │                │                 │
│ 강의 수강      │ 강의 개설       │ 전체 현황 관리   │
│ 퀴즈 응시      │ 자료 업로드     │ 학습 분석       │
│ 과제 제출      │ 퀴즈 생성       │ 이탈 위험 감지  │
│ AI 튜터 이용   │ 과제 출제       │ 강사 품질 관리  │
│ 학습 경로 확인 │ 수강생 모니터링 │                 │
└───────────────┴────────────────┴─────────────────┘
```

### 페르소나 1 — 학생 "김지수 (22세, 컴퓨터공학과 2학년)"
- **상황:** 학기 중 여러 과목을 동시에 수강하며 복습 시간 부족
- **니즈:** 수업 후 모르는 내용을 즉시 해결하고 싶음. 내 수준에 맞는 학습 경로 원함
- **Pain Point:** 교수님께 질문하기 어렵고, 스스로 무엇을 더 공부해야 할지 모름
- **LearnAI 활용:** AI 튜터로 24시간 질문, 개인화 학습 경로 추천 받기

### 페르소나 2 — 교사 "이민준 (35세, 프리랜서 강사)"
- **상황:** 온라인으로 Python 기초 강의 운영 중, 수강생 30명
- **니즈:** 매주 자료에 맞는 퀴즈 출제가 부담. 코딩 과제 채점 시간 단축 원함
- **Pain Point:** 퀴즈 1세트 만드는 데 2~3시간 소요. 개별 피드백 불가능
- **LearnAI 활용:** AI 퀴즈 자동 생성, 코딩 과제 AI 자동 피드백

### 페르소나 3 — 운영자 "박서연 (40세, 교육 플랫폼 매니저)"
- **상황:** 교육 기관에서 수십 개 강의, 수백 명 수강생 관리
- **니즈:** 어느 학생이 이탈 위험인지 빠르게 파악하고 개입하고 싶음
- **Pain Point:** 개별 수강 데이터 확인이 번거롭고, 이탈 후에야 알게 됨
- **LearnAI 활용:** 통합 대시보드, 이탈 위험 학생 자동 감지

## 5. 핵심 기능 설계

### 5-1. 교사(Teacher) 기능

| 기능 | 상세 내용 |
|------|-----------|
| **강의 관리** | 강의 개설(초안/공개/보관 상태 관리), 카테고리 분류, 수정·삭제 |
| **주차별 학습 자료 업로드** | PDF, PPT/PPTX, 이미지, 텍스트, 영상 지원. 드래그 앤 드롭 방식. Supabase Storage 저장 |
| **AI 퀴즈 자동 생성** | 업로드 자료 분석 또는 주제 입력 기반. 난이도(쉬움/보통/어려움) 선택. 3~10문항 설정. Claude AI가 객관식 문제·정답·해설 자동 생성 |
| **코딩 과제 출제** | Python / C / Java 언어 지원. 마감일 설정. 문제 지문 작성 |
| **수강생 모니터링** | 진도율, 퀴즈 평균 점수, 과제 제출 현황 실시간 확인 |
| **메시지 관리** | 학생 질문 확인 및 답변 |
| **공지사항** | 강의별 공지사항 등록 |
| **강사 평점 확인** | 완강 학생이 남긴 별점·리뷰 확인 |

### 5-2. 학생(Student) 기능

| 기능 | 상세 내용 |
|------|-----------|
| **강의 수강 신청** | 공개된 강의 목록 조회 및 수강 신청 |
| **통합 학습 뷰** | 주차별 자료·퀴즈·과제를 한 화면에서 확인 |
| **코드 에디터** | Monaco Editor 기반 브라우저 내 코드 작성·실행(Wandbox API). AI 피드백 즉시 수령 |
| **퀴즈 응시** | 객관식 퀴즈 풀기, 즉각적 정답·해설 피드백 |
| **AI 튜터 채팅** | 강의 맥락 기반 실시간 스트리밍 답변(Claude API). 대화 이력 자동 저장 |
| **AI 개인화 학습 경로** | 수강 진도·퀴즈 성적·과제 현황 분석 → 맞춤형 학습 로드맵 생성 |
| **학습 캘린더** | 과제 마감일, 퀴즈 일정 캘린더 뷰 |
| **메시지** | 교사에게 1:1 질문 메시지 전송 |
| **강의 평가** | 완강 후 교사 별점·리뷰 작성 |
| **자료 요약** | 학습 자료 AI 요약 기능 |

### 5-3. 운영자(Admin) 기능

| 기능 | 상세 내용 |
|------|-----------|
| **통합 대시보드** | 전체 수강생·강의·강사 수, AI 활용 현황, 수료율 한눈에 확인 |

| **강의별 현황** | 강의명·교사·수강생 수·상태 테이블 뷰 |
| **학습 분석** | 수강생별 상세 분석 리포트 |
| **강사 품질 모니터링** | 강사별 평점·수료율 분석 |
| **마케팅 어드바이저** | AI 기반 마케팅 전략 추천 |

### 인증 및 권한 흐름

```
회원가입 → 역할 선택(학생/교사/운영자)
         → Supabase Auth 계정 생성
         → profiles 테이블 자동 생성 (트리거)
         → 역할별 대시보드 리다이렉트

미들웨어(middleware.ts)
  → 모든 요청에서 세션 검증
  → 비인증 사용자 /login 리다이렉트
  → 역할 불일치 시 권한 오류

RLS(Row Level Security)
  → DB 쿼리 수준에서 역할별 데이터 접근 제어
  → 학생은 본인 데이터만 / 교사는 본인 강의 데이터만
```

### 주요 사용자 여정 (User Journey)

**학생의 전형적 학습 흐름:**
```
로그인 → 대시보드 확인 (진도·퀴즈 통계)
       → 강의 둘러보기 → 수강 신청
       → 강의 상세 → 주차별 자료 열람
       → AI 튜터로 질문 → 스트리밍 답변 수신
       → 퀴즈 응시 → 즉각 피드백
       → 코딩 과제 → Monaco 에디터 작성
       → 실행(Wandbox) → AI 피드백 수령
       → 학습 경로 확인 → 다음 강의 추천
```

**교사의 강의 운영 흐름:**
```
로그인 → 대시보드 (수강생·진도·평점 현황)
       → 강의 생성 → 자료 업로드 (드래그 앤 드롭)
       → AI 퀴즈 생성 (자료 선택 → Claude 분석 → 자동 생성)
       → 코딩 과제 출제 → 마감일 설정
       → 수강생 제출 현황 모니터링
       → 메시지 답변
```

## 9. AI 기능 상세 설계

LearnAI의 핵심 차별점은 **6가지 AI 기능**의 교육 특화 설계입니다.

### 9-1. AI 퀴즈 자동 생성

```
입력:
  - 강의 ID + 주차 + 난이도(쉬움/보통/어려움) + 주제
  - 선택: 학습 자료 파일 (텍스트 직접 분석)

처리 (Claude API):
  - 자료 있을 경우: 파일 내용 추출 → Claude에 전달 → 자료 기반 문제 생성
  - 자료 없을 경우: 주제만으로 문제 생성
  - 출력 형식: JSON [{question, options:[4개], answer, explanation}, ...]

저장:
  - quizzes 테이블에 저장 (JSONB 타입)
  - 수강생은 수강 중인 강의의 퀴즈만 열람 가능 (RLS)
```

### 9-2. AI 튜터 채팅 (실시간 스트리밍)

```
시스템 프롬프트 설계:
  - 강의명 + 소개 주입
  - 강의 텍스트 자료 최대 2개 (각 3,000자) 컨텍스트 주입
  - 역할: "${강의명}" 강의의 AI 튜터

대화 이력 관리:
  - chat_messages 테이블에 user/assistant 메시지 저장
  - 매 요청마다 최근 20개 이력 불러와 Claude에 전달
  - 맥락 연속성 보장

스트리밍:
  - ReadableStream + Server-Sent Events
  - Claude의 text_delta 이벤트를 실시간 클라이언트 전송
  - 완성 후 DB 저장
```

### 9-3. AI 개인화 학습 경로 분석

```
분석 데이터 수집:
  - 수강 중인 강의 + 진도율 + 상태
  - 퀴즈 결과 (퀴즈별 최고 점수, 난이도)
  - 과제 제출 현황 (제출/미제출)
  - 미수강 가능 강의 목록

Claude 분석 출력 형식:
  ## 현재 학습 진단
  ## 잘 하고 있는 점
  ## 집중해야 할 부분
  ## 추천 학습 경로 (1주/2주/한 달)
  ## 추천 강의
  ## 학습 팁
```

### 9-4. 코딩 과제 AI 자동 피드백

```
단계:
  1. Wandbox API로 코드 실행 (Python/C/Java)
     - 실행 결과(stdout), 오류(stderr), 상태 수집

  2. Claude API로 AI 피드백 생성
     입력: 문제 지문 + 제출 언어 + 학생 코드 + 실행 결과
     출력:
       ### 전반적인 평가
       ### 잘한 점
       ### 개선할 점
       ### 힌트 (정답 직접 제공 금지)

  3. DB 저장
     - assignment_submissions 테이블
     - 코드, 실행 결과, AI 피드백, 시도 횟수 모두 저장
```

### 9-5. 학습 자료 AI 요약

```
- 업로드된 자료를 Claude가 핵심 내용으로 요약
- 학생이 긴 자료를 빠르게 파악하는 데 활용
- /api/summarize 엔드포인트 처리
```

### 9-6. AI 마케팅 어드바이저 (운영자용)

```
- 운영자가 현황을 입력하면 AI가 마케팅 전략 추천
- 수강생 유치, 강의 홍보 방향 등 조언
- /api/marketing-advice 엔드포인트 처리
```

## 10. 기술 스택 선정 이유

| 기술 | 선정 이유 |
|------|-----------|
| **Next.js 16 (App Router)** | SSR로 초기 로딩 속도 최적화, API Routes로 백엔드 통합, Server Actions로 폼 처리 간소화 |
| **TypeScript** | 타입 안전성으로 런타임 오류 예방, 대규모 코드베이스 유지보수성 향상 |
| **Tailwind CSS v4** | 빠른 UI 개발, 다크모드 지원, 커스텀 디자인 시스템 구축 용이 |
| **Supabase** | PostgreSQL + Auth + Storage + RLS를 하나로 제공. 실시간 기능, 확장성, 무료 티어 |
| **Anthropic Claude API** | 한국어 이해 탁월, 교육용 답변 품질 우수, claude-opus-4-6 최신 모델 활용 |
| **Wandbox API** | 서버리스 환경에서 안전한 코드 실행. Python/C/Java 지원. 별도 샌드박스 불필요 |
| **Monaco Editor** | VS Code와 동일한 코드 에디터, 문법 강조·자동완성 지원 |
| **Vercel** | Next.js 최적화 배포, 글로벌 CDN, 자동 HTTPS, CI/CD 통합 |

### 아키텍처 결정 원칙

1. **서버리스 우선:** 별도 서버 없이 Vercel + Supabase로 완전한 풀스택 구현
2. **보안 내재화:** RLS로 DB 레벨에서 권한 제어, 미들웨어로 라우트 보호
3. **스트리밍 UX:** AI 응답을 스트리밍으로 전송하여 체감 속도 향상
4. **역할 기반 분리:** URL 구조 자체로 역할 분리 (`/student/`, `/teacher/`, `/admin/`)
